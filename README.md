# 왜 스프링인가?
지극히 개인적이고 주관적인 제 생각입니다. 

- 지금껏 배워 왔던 Flask는 너무 자유분방적이었다고 생각합니다. 자유도가 높은 만큼 자신만의 구조를 만들고 틀을 짤 수 있었지만 제게는 너무 어렵게만 느껴졌습니다. 주변 분들이 만들어두신 구조를 따라하고 익히는 것보다 이미 공식적으로 가이드라인이 있고, 방대한 공식 문서가 있는 스프링을 공부하고 싶었습니다.
- 라이브러리 관리가 힘들었다고 느꼈습니다. 실행 환경이 달라질 때마다 라이브러리를 관리하는 것이 제게는 저의 리소스를 필요 없는 곳에 사용하는 것 같아 많이 아쉬웠습니다. 허나 스프링은 의존성을 관리함으로써 제가 얻을 수 있는 편리함이 매우 크기에 리소스를 효율적으로 활용할 수 있을 것 같습니다.
- 니즈가 많았고, 커뮤니티도 많이 활성화되어 있습니다. 모르는 것이 있으면 커뮤니티의 회원분들께서는 친절히 설명해주셨는데, 공부하는 데에 많은 도움이 되었습니다.
- 방대한 양의 라이브러리 지원: 평소 필요한 라이브러리가 있으면 어느 것을 사용해야 하는지, 그 라이브러리를 내가 직접 구현하지 않고 이용하면 무엇을 얻을 수 있는지 등 여러 생각을 해 보아야 했습니다. 또한 사용하려는 라이브러리의 신뢰성 또한 따져보아야 했기에 아쉬웠던 점이 조금 있었습니다.
